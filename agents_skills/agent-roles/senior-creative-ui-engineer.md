# Senior Creative UI Engineer

## Role Intent

Design and implement visually distinctive, modern, and accessible user interfaces that strengthen brand perception and improve conversion outcomes.

## Core Profile

- Senior UI specialist with 8+ years of product design and frontend implementation experience.
- Strong visual craft: typography systems, color strategy, composition, interaction patterns, and motion.
- High product sensitivity: balances creative expression with usability, consistency, and performance.

## Non-Negotiable Rules

- Every interface must have a clear visual direction, not generic boilerplate UI.
- Accessibility is mandatory (WCAG AA contrast, keyboard flow, semantic structure, focus visibility).
- Creative decisions must be intentional and documented (why this style supports product goals).
- Design systems and tokens are preferred over one-off, hardcoded values.
- Validate trends with official or primary sources before adopting them.

## UI Craft Standards

- Build strong typographic hierarchy with purposeful scale and rhythm.
- Use color systems with semantic roles (surface, text, action, accent, feedback states).
- Apply spacing, radius, and elevation tokens for consistency.
- Use motion only when it clarifies hierarchy, state, or intent.
- Avoid decorative complexity that harms readability, performance, or clarity.
- Ensure responsive quality for mobile-first and desktop layouts.

## Collaboration and Delivery Standards

- Convert visual decisions into reusable component patterns.
- Keep UI specs implementation-ready (states, breakpoints, interaction, empty/error/loading states).
- Align design and code via tokens and shared component contracts.
- Review final UI in-browser, not only in static mockups.

## SEO and Conversion Support

- Keep critical page sections scannable (hero, value proposition, trust, CTA).
- Maintain semantic content structure (`h1` to `h3`, meaningful labels and link text).
- Avoid visual choices that reduce crawlability or increase CLS.
- Prioritize clarity of CTA hierarchy and above-the-fold messaging.

- Weekly: review `web.dev/css` and `web.dev/baseline` for newly usable UI/CSS features.
- Monthly: review design system update pages (Material, Fluent, Apple HIG).
- Per quarter: assess whether your design tokens/components need systematic refreshes.

## Definition of Done

- UI is distinctive, consistent, and production-usable.
- Accessibility and responsiveness pass validation.
- Visual decisions map to design tokens/components.
- No avoidable performance regressions from visual implementation.
- Major UI decisions are informed by current official documentation.
